1. Introducción

El objetivo de este taller es mostrar de una manera prática la utilización de git, sus comandos básicos y esenciales mediante desarrollo de un proyecto real:

Repositorio de github
WPZOOM Template

2. Metodología

El taller esta diseñado para realizar equipos de 3 personas, cada miembro debe seleccionar un rol estos son:

  1. Front End Web Developer
  2. Graphic Designer
  3. Front End Designer

3. Requerimientos

Para poder realizar este taller cada miembro del equipo debe cumplir los siguientes requisitos:

  1. Seleccionar el rol de cada miembro del equipo y es recomendable sentarse en el orden de los roles descritos anterioremente
  2. Tener una cuenta activa de github
  3. Instalar la última versión de git
  4. Conocimientos básicos del editor vi de linux.
  5. Seleccionar el directorio de trabajo base para el Taller
  6. Configurar el nombre completo y correo electrónico (correspondiente a la cuenta de github) en git con
    git config --global user.name "[Nombre y Apellido]"
    git config --global user.email [Correo electrónico]

4. Creación del repositorio

Uno de los miembros del equipo debe ser el encargado de crear el repositorio en github, el miembro del equipo que va a realizar esta tarea debe seguir los siguientes pasos:

  1. Ingresar a github
  2. En la parte superior derecha hacer clic en el enlace + y seleccionar la opción New repository
  3. En el campo Repository name digitar: academia y presionar el botón Create repository
  4. En el repositorio creado hacer clic en la parte derecha en el enlace Settings
  5. Hacer clic en la parte izquierda en el enlace Collaborators
  6. En el campo Type a username digite el nombre de usuario de otro miembro del equipo y presione el botón Add collaborator (Repetir este paso hasta que todos los miembros del equipo queden agregados al repositorio).
  7. Finalmente los otros miembros del equipo debe revisar su cuenta de correo asociada a la cuenta de github para aceptar la invitación realizada en el paso anterior.

En el taller sustituir [nombre-de-usuario] por el nombre de usuario del miembro del equipo que creo el repositorio
